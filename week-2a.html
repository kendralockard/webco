Text Fields and Forms

Sometimes we want the user's input. To do this, we will use .

But what good is the user's input if they can't submit it? Luckily, we can build web forms that submit data to a server using HTML alone. Just use the form element.

To make the UI a little friendlier, we can add a submit button for this text field. All we have to do is add a button as the last element of our form element.

Sometimes, we want to require that a user fill out a certain field. We can enforce this by adding a required attribute to our input element.

Buttons and Checkboxes

There are more features that may be useful in getting other types of user input. For example, buttons are a common tool used for multiple choice or checkbox type questions. First we will look at radio buttons, which are HTML's multiple choice button type.

All related radio buttons should have the same name attribute, to ensure that only one answer is chosen.
It is considered best practice to set a for attribute on label elements with a value that matches that of the id value of the input element. This allows assisitive technologies to create a linked relationship between the label, and the child input element.

Checkboxes are another HTML tool available to use for questions with several possible answers. Just like buttons, "checkbox" is a type of input. In fact, they work just the same as radio buttons, with the same conventions and best practices.
